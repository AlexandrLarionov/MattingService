# Matting Service (Background Remover)

Сервис для автоматического удаления фона (маттирования) с изображений, реализованный на базе **FastAPI**. Предоставляет REST API и веб-интерфейс для обработки изображений с использованием передовых SOTA (State of the Art) моделей. 

## Описание

Сервис решает задачу **маттирования** (предсказание alpha-канала с непрерывными значениями [0.0…1.0]), в отличие от простой бинарной сегментации. Это позволяет корректно обрабатывать сложные границы: волосы, мех, ворс, тонкий текстиль и полупрозрачные объекты. Наличие точного альфа-канала критично для последующего композитинга в GenAI-пайплайнах и e-commerce, так как предотвращает появление "halo" артефактов на краях.

## Поддерживаемые модели

В сервисе реализовано два режима обработки на базе разных архитектур, отобранных по результатам внутреннего тестирования.

### 1. BiRefNet (`mode="birefnet"`)
* **Тип:** Однопроходный маттинг (Bilateral Reference Network)
* **Назначение:** Универсальная и быстрая автоматическая обработка.
* **Особенности:** Стабильно и качественно работает на подавляющем большинстве доменов e-commerce. Архитектура строится на Swin Transformer с механизмом двусторонних референсов (глобальный контекст и локальные детали). Является основной ("рабочей лошадкой") моделью сервиса. Работает значительно быстрее двухэтапных аналогов.
* **Слабые места:** Может незначительно уступать на крайне сложных деталях (тонкие волосы/мех) или сильно прозрачных объектах.

### 2. SAM2Matting + BEN2 (`mode="ben2+sam2matting"`)
* **Тип:** Двухэтапный пайплайн (Детектор + Рефайнер)
* **Назначение:** Высокоточное маттирование для сложных случаев.
* **Особенности:** Разделение ответственности. На первом этапе BEN2 Base генерирует первичную маску (локализует объект). На втором этапе модель SAM2Matting (с замороженным VOS-трекером) через механизм ROI Detector и Progressive Alpha Predictor филигранно уточняет полупрозрачные зоны и тонкие структуры. 
* **Преимущества:** SOTA-качество на волосах, мехе и прозрачных объектах (стекло). Идеально плавные края и субпиксельная точность.

## Архитектура проекта

Проект организован в виде современного микросервиса на базе **FastAPI**.

```text
matting_service/
├── app/                      # Исходный код сервиса
│   ├── api/                  # Слой API (маршрутизация FastAPI, веб-интерфейс)
│   │   ├── main.py           # Точка входа в приложение
│   │   └── static/           # Статические файлы (HTML, CSS, JS)
│   ├── core/                 # Конфигурация и константы (пути)
│   └── pipelines/            # Бизнес-логика (обёртки над моделями)
│       ├── onnx_pipeline.py  # Пайплайн для BiRefNet (ONNX)
│       └── sam2_pipeline.py  # Пайплайн для SAM2 + BEN2
├── models_onnx/              # Директория с весами моделей (ONNX, PyTorch ckpt)
├── sam2/                     # Библиотечный код SAM2Matting (исходники)
└── Dockerfile                # Инструкция для сборки Docker-образа
```

## Особенности реализации (Engineering Highlights)

* **Потокобезопасный инференс (Thread-safety):** Вызовы моделей защищены с помощью `asyncio.Lock()`. Это предотвращает OOM (Out of Memory) и падения C++ бэкендов PyTorch/ONNX при одновременной отправке множества запросов к одной модели. При этом разные модели (BiRefNet и SAM2) могут обрабатывать запросы полностью параллельно.
* **Асинхронность (Non-blocking I/O):** Вся тяжелая CPU/GPU-нагрузка вынесена в `ThreadPoolExecutor` через `asyncio.to_thread()`, что позволяет Event Loop'у FastAPI (uvicorn) продолжать мгновенно принимать новые HTTP-запросы без блокировок.
* **Профессиональное логирование:** Использование `loguru` для структурированных, информативных и потокобезопасных логов.
* **Smart Preprocessing:** Встроенная корректировка ориентации (EXIF Orientation) для корректной работы с мобильными фотографиями.

## Как запустить

### Клонирование репозитория (Git LFS)

Поскольку веса нейросетей (файлы `.onnx` и `.pt`) весят достаточно много, в репозитории используется **Git LFS** (Large File Storage).

Для корректного скачивания моделей при клонировании убедитесь, что у вас установлен Git LFS:

1. Установите Git LFS (если еще не установлен):
```bash
git lfs install
```

2. Склонируйте репозиторий:
```bash
git clone https://github.com/AlexandrLarionov/MattingService.git
cd matting_service
```

3. Подтяните файлы моделей из LFS:
```bash
git lfs pull
```

### Локальный запуск (Development)

1. Установите зависимости (предполагается активированное виртуальное окружение с установленным PyTorch и CUDA):
```bash
pip install -r requirements.txt
```

2. Запустите uvicorn:
```bash
uvicorn app.api.main:app --host 0.0.0.0 --port 8000 --reload
```
Сервис будет доступен по адресу `http://localhost:8000`.

### Запуск через Docker

Соберите и запустите контейнер (убедитесь, что директория `models_onnx` наполнена необходимыми весами):
```bash
docker-compose up --build -d
```

## API Endpoint

### `POST /api/remove-bg`
Удаляет фон с загруженного файла.

**Формат запроса (multipart/form-data):**
* `file`: Файл изображения (JPG, PNG, WebP) до 20 МБ.
* `mode`: Режим обработки (`birefnet` или `ben2+sam2matting`). По умолчанию `birefnet`.

**Ответ:** 
Файл формата `PNG` с прозрачным фоном (RGBA) и кастомным заголовком `X-Processing-Time`, содержащим время инференса в секундах.
